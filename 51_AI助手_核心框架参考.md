## 核心参考框架

打造专属AI助手可以理解为"组装高性能电脑"：基于强大的核心框架，按需组装记忆系统、模型部署等多个组件。

### 主要参考框架

| 框架 | 定位 | 特点 | 地址 |
|------|------|------|------|
| **Nanobot** | 主要参考 | 极简、高可读性，约4000行代码，非常适合学习和二次开发。Python原生，低资源消耗（1GB内存可运行）。已支持MCP (Model Context Protocol) 接口。 | [GitHub: HKUDS/nanobot](https://github.com/HKUDS/nanobot) |
| **OpenClaw** | 功能蓝图 | 功能全面的自托管AI助手，可参考其Skills系统和多Agent协作模式。 | [GitHub: openclaw/openclaw](https://github.com/openclaw/openclaw) |
| **Hermes Agent** | 架构参考 | 有状态、可自进化的智能体运行时，强调长期记忆和用户建模。 | - |

### Python Agent框架

| 框架 | 适用场景 | 说明 |
|------|----------|------|
| **LangChain / LangGraph** | 首选方案 | 最主流LLM应用框架，提供模型I/O、检索、链、Agent、记忆等标准化组件。LangGraph适合复杂多步骤Agent。 |
| **AgentForge** | 轻量选择 | 学术项目，强调模块化和可组合性，避免供应商锁定。 |

---

## 本地化部署

| 工具 | 用途 | 说明 |
|------|------|------|
| **Ollama** | 必备工具 | 一键运行和管理本地开源模型（Llama 3, Qwen, Mistral），提供OpenAI API兼容接口 |
| **llama-cpp-python** | 底层依赖 | 高效运行GGUF格式模型，许多本地部署方案的底层依赖 |

---

## 记忆与知识库

| 方案 | 类型 | 说明 |
|------|------|------|
| **Mem0** | 记忆层 | 为Agent提供跨会话的用户、对话和上下文记忆 |
| **LangChain Memory** | 短期记忆 | `ConversationBufferMemory`等，管理当前会话上下文 |
| **Chroma** | 向量数据库 | 轻量级，适合构建RAG知识库 |

## 快速上手路线

1. **环境准备**: Python 3.9-3.11，创建虚拟环境
2. **从Nanobot开始**: `pip install nanobot-ai`，先跑起来感受架构
3. **学习LangChain**: 实现简单的工具调用Agent
4. **组合核心组件**:
   - LangChain 作为应用骨架
   - Ollama 运行本地模型
   - Mem0 管理记忆
   - Chroma 构建知识库
   - 自定义工具扩展功能
5. **功能迭代**: 逐步加入多Agent协作、用户建模等进阶功能

## 深入了解：框架对比

### Nanobot vs OpenClaw vs Hermes

| 维度 | Nanobot | OpenClaw | Hermes Agent |
|------|---------|----------|--------------|
| 代码量 | ~4000行 | 成熟庞大 | 中等 |
| 复杂度 | 极简 | 复杂 | 中等 |
| 核心特点 | Gateway架构 + MCP支持 | Skills系统 + 多Agent | 用户建模 + 自进化引擎 |
| 资源需求 | 极低 | 较高 | 中等 |
| 学习曲线 | 平缓 | 陡峭 | 中等 |

### Nanobot 架构亮点
- 统一的Gateway架构，连接不同消息平台（Discord, Slack, Telegram）
- MCP服务器支持（标准化扩展接口）
- 1GB内存可运行，适合边缘设备

### Hermes Agent 可借鉴思想
- **用户建模层 (Honcho)**：长期学习用户偏好
- **自进化引擎**：在长期运行中持续优化
