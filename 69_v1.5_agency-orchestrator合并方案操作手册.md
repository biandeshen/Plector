# Plector + Agency Orchestrator + Claude Code 全栈集成 v1.5

> **合并方案**：取 v1 MCP 集成 + v2 业务封装，两层级联
>
> 目标：用户一句话需求 → auto_developer 一键调度 → agency-orchestrator 编排 174 个角色 → Claude Code 执行 → 输出完整代码
>
> 涉及组件：Plector（总控）+ agency-orchestrator MCP Server（编排）+ agency-agents-zh（角色）+ superpowers-zh（方法论）+ Claude Code（执行）

---

## 方案架构

```
用户: "给用户模块加批量导出功能"
  │
  ├─ 方式A（一键开发）: auto_developer.develop()
  │    └→ 内部调用 agency-orchestrator MCP 的 compose_workflow + run_workflow
  │
  ├─ 方式B（分步控制）: 直接调用 agency-orchestrator MCP 工具
  │    ├→ list_workflows  → 选择模板
  │    ├→ compose_workflow → 生成 YAML
  │    ├→ plan_workflow    → 查看 DAG
  │    └→ run_workflow     → 执行
  │
  └─ 方式C（低层查询）: 直接调用 agency-orchestrator MCP 工具
       ├→ list_roles  → 查看角色
       └→ get_role    → 角色详情
```

**两层级联设计**：

| 层级 | 技能 | 职责 | 面向 |
|------|------|------|------|
| L1 | `agency_orchestrator` | MCP Server 直接暴露的 9 个工具 | 想精细控制的用户 |
| L2 | `auto_developer` | 一键开发封装（develop/compose/run） | 想要开箱即用的用户 |

L2 调用 L1，不重复造轮子。

# skills/agency_orchestrator/implementation.py
"""
Agency Orchestrator 技能实现（L1 — MCP Server 直接封装）

本技能通过 MCP Server (agency-orchestrator) 暴露 7 个工具。
实际执行由 Node.js MCP Server 完成，Python 侧提供：
1. 本地只读工具（list_roles, get_role, list_workflows — 直接读文件）
2. MCP 代理工具（run_workflow, compose_workflow 等 — 标记需 MCP 调用）
"""

import re
from pathlib import Path

SKILL_DIR = Path(__file__).parent
ROLES_DIR = SKILL_DIR.parent.parent / "external-skills" / "roles"
WORKFLOWS_DIR = SKILL_DIR.parent.parent / "servers" / "agency-orchestrator" / "workflows"


# ─── 本地只读工具 ───


def list_roles(category: str | None = None) -> dict:
    """列出所有 AI 角色，按分类。参数: category — 可选分类过滤"""
    if not ROLES_DIR.exists():
        return {"success": False, "error": f"角色目录不存在: {ROLES_DIR}"}

    result = {}
    for cat_dir in sorted(ROLES_DIR.iterdir()):
        if not cat_dir.is_dir():
            continue
        if category and cat_dir.name != category:
            continue
        result[cat_dir.name] = _scan_role_files(cat_dir)

    return {
        "success": True,
        "categories": list(result.keys()),
        "roles": result,
        "total": sum(len(v) for v in result.values()),
    }


def _scan_role_files(cat_dir: Path) -> list[str]:
    """扫描分类目录下的角色文件名"""
    roles = []
    for f in sorted(cat_dir.iterdir()):
        if f.suffix == ".md":
            roles.append(f.stem.replace(f"{cat_dir.name}-", ""))
    return roles


def get_role(category: str, name: str) -> dict:
    """获取指定角色的完整定义。参数: category — 分类, name — 角色名"""
    role_file = _find_role_file(category, name)
    if not role_file:
        return {"success": False, "error": f"角色不存在: {category}/{name}"}

    content = role_file.read_text(encoding="utf-8")
    fm_name, fm_desc = _parse_frontmatter(content)

    return {
        "success": True,
        "category": category,
        "name": fm_name or name,
        "description": fm_desc,
        "content": content,
    }


def _find_role_file(category: str, name: str) -> Path | None:
    """查找角色文件，尝试带前缀和不带前缀两种文件名"""
    candidates = [
        ROLES_DIR / category / f"{name}.md",
        ROLES_DIR / category / f"{category}-{name}.md",
    ]
    for p in candidates:
        if p.exists():
            return p
    return None


def _parse_frontmatter(content: str) -> tuple[str, str]:
    """解析 YAML frontmatter，返回 (name, description)"""
    fm_match = re.match(r"^---\n(.*?)\n---", content, re.DOTALL)
    if not fm_match:
        return ("", "")

    try:
        import yaml
        fm = yaml.safe_load(fm_match.group(1))
        return (fm.get("name", ""), fm.get("description", ""))
    except Exception:
        return ("", "")


def list_workflows() -> dict:
    """列出所有可用的工作流模板"""
    if not WORKFLOWS_DIR.exists():
        return {"success": False, "error": f"工作流目录不存在: {WORKFLOWS_DIR}"}

    templates = _scan_workflow_files()
    return {
        "success": True,
        "count": len(templates),
        "templates": templates,
    }


def _scan_workflow_files() -> list[dict]:
    """递归扫描工作流目录，提取 name 和 description"""
    import yaml as _yaml
    templates = []

    for f in sorted(WORKFLOWS_DIR.rglob("*.yaml")):
        rel = f.relative_to(WORKFLOWS_DIR)
        try:
            doc = _yaml.safe_load(f.read_text(encoding="utf-8"))
            templates.append({
                "name": str(rel).replace("\\", "/"),
                "desc": doc.get("description", doc.get("name", "")),
            })
        except Exception:
            templates.append({"name": str(rel).replace("\\", "/"), "desc": "(解析失败)"})

    return templates


# ─── MCP 代理工具（标记由 MCP Server 执行） ───


def run_workflow(
    path: str,
    inputs: dict | None = None,
    provider: str = "claude-code",
    model: str | None = None,
    resume: str | None = None,
    from_step: str | None = None,
) -> dict:
    """
    执行 YAML 工作流（由 MCP Server 执行）。

    参数:
        path: 工作流 YAML 文件路径
        inputs: 输入变量 { "key": "value" }
        provider: LLM 提供商（默认 claude-code，免 API key）
        model: 模型名称
        resume: 断点续跑 "last" 或目录路径
        from_step: 从指定步骤重新执行
    """
    return {
        "_mcp_call": "agency-orchestrator",
        "tool": "run_workflow",
        "args": {
            "path": path,
            "inputs": inputs or {},
            "provider": provider,
            **({"model": model} if model else {}),
            **({"resume": resume} if resume else {}),
            **({"from_step": from_step} if from_step else {}),
        },
    }


def validate_workflow(path: str) -> dict:
    """校验 YAML 工作流文件（由 MCP Server 执行）。参数: path — 工作流文件路径"""
    return {
        "_mcp_call": "agency-orchestrator",
        "tool": "validate_workflow",
        "args": {"path": path},
    }


def plan_workflow(path: str) -> dict:
    """显示 DAG 执行计划（由 MCP Server 执行）。参数: path — 工作流文件路径"""
    return {
        "_mcp_call": "agency-orchestrator",
        "tool": "plan_workflow",
        "args": {"path": path},
    }


def compose_workflow(
    description: str,
    provider: str = "claude-code",
    model: str | None = None,
) -> dict:
    """
    用自然语言生成 YAML 工作流（由 MCP Server 执行）。

    参数:
        description: 一句话需求描述
        provider: LLM 提供商（默认 claude-code）
        model: 模型名称
    """
    return {
        "_mcp_call": "agency-orchestrator",
        "tool": "compose_workflow",
        "args": {
            "description": description,
            "provider": provider,
            **({"model": model} if model else {}),
        },
    }


# 工具注册
TOOLS = {
    "run_workflow": run_workflow,
    "validate_workflow": validate_workflow,
    "list_workflows": list_workflows,
    "plan_workflow": plan_workflow,
    "compose_workflow": compose_workflow,
    "list_roles": list_roles,
    "get_role": get_role,
}

"""
auto_developer 技能实现（L2 — 一键开发封装）

调用 agency_orchestrator (L1) 的 MCP 工具，提供高层一键开发体验。
内部流程：compose_workflow 生成 YAML → run_workflow 执行 → 读取结果
"""

from pathlib import Path

SKILL_DIR = Path(__file__).parent
PROJECT_DIR = SKILL_DIR.parent.parent
AO_OUTPUT_DIR = PROJECT_DIR / "ao-output"

# 默认工作流（一键开发时使用）
DEFAULT_WORKFLOW = PROJECT_DIR / "workflows" / "auto_develop.yaml"


def develop(
    requirement: str,
    project_dir: str = ".",
    provider: str = "claude-code",
) -> dict:
    """
    一键自动开发：输入需求，自动执行完整开发流水线。

    参数:
        requirement: 开发需求描述
        project_dir: 项目目录路径
        provider: LLM 提供商（默认 claude-code，免 API key）
    """
    if not DEFAULT_WORKFLOW.exists():
        return {
            "success": False,
            "error": f"默认工作流不存在: {DEFAULT_WORKFLOW}",
            "hint": "请先运行 compose 创建工作流，或检查 workflows/auto_develop.yaml",
        }

    return {
        "_mcp_call": "agency-orchestrator",
        "tool": "run_workflow",
        "args": {
            "path": str(DEFAULT_WORKFLOW),
            "inputs": {
                "requirement": requirement,
                "project_dir": project_dir,
            },
            "provider": provider,
        },
        "_post_process": "read_summary",
    }


def compose(description: str, provider: str = "claude-code") -> dict:
    """
    根据描述自动生成工作流 YAML。

    参数:
        description: 工作流描述
        provider: LLM 提供商（默认 claude-code）
    """
    return {
        "_mcp_call": "agency-orchestrator",
        "tool": "compose_workflow",
        "args": {
            "description": description,
            "provider": provider,
        },
    }


def run(
    workflow: str,
    inputs: dict | None = None,
    provider: str = "claude-code",
) -> dict:
    """
    运行指定的工作流。

    参数:
        workflow: 工作流 YAML 文件路径
        inputs: 输入变量字典
        provider: LLM 提供商（默认 claude-code）
    """
    return {
        "_mcp_call": "agency-orchestrator",
        "tool": "run_workflow",
        "args": {
            "path": workflow,
            "inputs": inputs or {},
            "provider": provider,
        },
    }


def plan(workflow: str) -> dict:
    """
    查看工作流的 DAG 执行计划。

    参数:
        workflow: 工作流 YAML 文件路径
    """
    return {
        "_mcp_call": "agency-orchestrator",

def read_latest_summary() -> str | None:
    """读取最新的 ao-output 输出摘要"""
    if not AO_OUTPUT_DIR.exists():
        return None

    dirs = sorted(
        [d for d in AO_OUTPUT_DIR.iterdir() if d.is_dir()],
        key=lambda d: d.stat().st_mtime,
        reverse=True,
    )

    if not dirs:
        return None

    summary_file = dirs[0] / "summary.md"
    if summary_file.exists():
        return summary_file.read_text(encoding="utf-8")

    return None

`workflows/auto_develop.yaml`：

```yaml
name: "自动开发流水线"
description: "从需求分析到代码实现的全自动开发流程"

agents_dir: "agency-agents-zh"

llm:
  provider: "claude-code"
  model: "claude-sonnet-4-20250514"
  max_tokens: 8192

concurrency: 2

inputs:
  - name: requirement
    description: "开发需求描述"
    required: true
  - name: project_dir
    description: "项目目录路径"
    required: true

steps:
  - id: analyze
    role: "product/product-manager"
    task: |
      分析以下开发需求，输出：
      1. 功能需求清单
      2. 非功能性需求
      3. 验收标准
      4. 优先级排序

      需求：{{requirement}}
    output: requirements

  - id: design
    role: "engineering/engineering-software-architect"
    task: |
      基于以下需求，设计技术方案：
      1. 接口设计（API 端点、请求/响应格式）
      2. 数据库变更（表结构、索引）
      3. 核心逻辑流程
      4. 异常处理策略

      需求：{{requirements}}
    output: tech_design
    depends_on: [analyze]

  - id: security
    role: "engineering/engineering-security-engineer"
    task: |
      审查以下技术方案的安全性：
      1. 权限校验是否完善
      2. 数据泄露风险
      3. 注入攻击防护
      4. 敏感数据处理

      方案：{{tech_design}}
    output: security_review
    depends_on: [analyze]

  - id: implement
    role: "engineering/engineering-senior-developer"
    task: |
      根据以下技术方案和安全审查意见，在项目目录中实现代码：
      1. 严格按照技术方案实现
      2. 遵循安全审查意见
      3. 代码风格遵循项目现有规范

      项目目录：{{project_dir}}
      技术方案：{{tech_design}}
      安全意见：{{security_review}}
    output: implementation
    depends_on: [design, security]

  - id: review
    role: "engineering/engineering-code-reviewer"
    task: |
      审查以下实现代码：
      1. 代码质量和可维护性
      2. 是否符合技术方案
      3. 是否遵循安全意见
      4. 测试覆盖是否充分

      实现：{{implementation}}
    output: code_review
    depends_on: [implement]

  - id: summary
    role: "product/product-manager"
    task: |
      汇总开发结果：
      1. 实现了哪些功能
      2. 代码审查意见
      3. 后续改进建议

      需求：{{requirements}}
      实现：{{implementation}}
      审查：{{code_review}}
    depends_on: [review]
```
