## 二、P0 — 必须立即修复（安全漏洞）

### P0-1：test_runner 命令注入

- **文件**：`skills/test_runner/implementation.py` L100-102
- **问题**：`subprocess.run(command, shell=True)` 接受任意字符串，LLM 可被诱导执行恶意命令
- **影响**：RCE（远程代码执行）
- **修复**：
  ```python
  # 1. run_command 改为列表模式
  result = subprocess.run(
      shlex.split(command),  # 安全拆分
      shell=False,           # 禁用 shell
      capture_output=True, text=True, timeout=timeout
  )
  # 2. run_tests 固定命令，path 做白名单校验
  SAFE_PATH = re.match(r'^[a-zA-Z0-9_/.-]+$', path)
  ```

### P0-2：web_search SSRF 风险

- **文件**：`skills/web_search/implementation.py` L96-98
- **问题**：`fetch_page()` 无 URL 白名单/内网过滤，可请求 `http://169.254.169.254`（AWS 元数据）
- **影响**：云环境凭证泄露
- **修复**：复用 `core/image_handler.py` 的 SSRF 防护模块（`_is_private_ip`、DNS pinning）

### P0-3：code_writer 路径穿越

- **文件**：`skills/code_writer/implementation.py`
- **问题**：`write_code()`、`read_code()`、`modify_code()` 无路径安全校验，可写 `/etc/passwd`
- **影响**：任意文件读写
- **修复**：添加项目根目录白名单
  ```python
  PROJECT_ROOT = Path(__file__).parent.parent.parent.resolve()
  def _check_safe_path(self, path: Path):
      if not str(path.resolve()).startswith(str(PROJECT_ROOT)):
          raise PermissionError(f"禁止操作项目外路径: {path}")
  ```

### P0-4：.env API Key 泄露风险

- **文件**：`.env`
- **问题**：虽然 `.gitignore` 已排除 `.env`，但需检查 Git 历史是否曾提交
- **修复**：
  ```bash
  git log --all --full-history -- .env
  # 如有历史记录，需用 git filter-branch 或 BFG 清除
  ```

## 三、P1 — 尽快修复（稳定性/架构）

### P1-1：mcp_client.py 与 mcp_manager.py 重叠

- **问题**：两者都做 MCP 连接管理，MCPManager 每 Server 创建独立 MCPClient（L62）
- **修复**：删除 `mcp_manager.py`，将其 `.env` 加载逻辑移入 `config_loader.py`，所有调用改用 `MCPClient`

### P1-2：硬编码 Windows 路径

| 文件 | 行 | 硬编码内容 |
|------|-----|-----------|
| `mcp_client.py` | L69 | `r"C:\Users\dev\.local\bin"` |
| `config.yaml` | L63 | `C:\\Users\\dev\\.local\\bin\\uvx.exe` |
| `agent_loop.py` | L137 | `sqlite3.connect("data/plector.db")` |
| `vector_memory.py` | L33 | `DB_PATH = "data/vector_memory"` |

- **修复**：全部改为 `os.path` 或 `Path` 动态解析 + config.yaml 参数化
  ```python
  # mcp_client.py
  uv_path = os.path.expanduser("~/.local/bin")
  # config.yaml — 支持环境变量
  command: "${UVX_PATH:-uvx}"
  ```

### P1-3：stdio 读取无限循环

- **文件**：`mcp_client.py` L209-222
- **问题**：跳过非 JSON 行和等待 `id` 字段的 while 循环无超时，可能永久阻塞
- **修复**：添加 `asyncio.wait_for` 超时
  ```python
  response_line = await asyncio.wait_for(
      self.process.stdout.readline(), timeout=30.0
  )
  ```

### P1-4：EventBus 通配符 Bug + fire-and-forget

- **文件**：`event_bus.py`
- **问题**：
  1. `pattern.endswith("*")` 会匹配自身（`"skill.*"` 的 `pattern[:-1]` = `"skill"` → `"skill".startswith("skill")` = True）
  2. `asyncio.create_task()` 无错误追踪
- **修复**：
  ```python
  # 1. 修复通配符
  if pattern.endswith(".*") and event_type.startswith(pattern[:-2] + "."):
  # 2. 错误追踪
  async def _safe_handler(handler, event):
      try:
          await handler(event)
      except Exception as e:
          logger.error(f"Event handler 异常: {e}", exc_info=True)
  asyncio.create_task(_safe_handler(handler, event))
  ```

### P1-5：governance.py 未实现

- **问题**：`_detect_cycles()` 返回 `[]`，`auto_eliminate()` 只有 `pass`
- **修复**：实现基于 DFS 的环检测 + 健康分数淘汰事件发布

### P1-6：LLM 客户端每次新建连接

- **文件**：`llm_client.py` L33
- **问题**：`_ollama_chat` 每次创建新 `AsyncClient`，无连接复用
- **修复**：构造函数中初始化，惰性创建

### P1-7：Anthropic 多 system 消息丢失

- **文件**：`llm_client.py` L88-93
- **问题**：多条 system 消息只保留最后一条
- **修复**：用 `"\n".join()` 合并所有 system 消息

## 四、P2 — 计划修复（代码质量/兼容性）

### P2-1：image_handler._stream_check_size 异常返回 True

- **文件**：`image_handler.py` L443
- **问题**：异常时返回 `(True, "")` — 下载失败仍视为"安全通过"
- **修复**：改为 `return (False, "流式检查失败")`

### P2-2：function_calling.py Schema 变异

- **文件**：`function_calling.py` L23-32
- **问题**：`register()` 直接修改传入的 `input_schema` 字典，污染调用方数据
- **修复**：`input_schema = copy.deepcopy(input_schema)`

### P2-3：file_utils Windows 路径检查缺陷

- **文件**：`file_utils/implementation.py` L264
- **问题**：`resolved.startswith(forbidden + "/")` 只检查正斜杠
- **修复**：
  ```python
  sep = os.sep
  if resolved == forbidden or resolved.startswith(forbidden + sep) or resolved.startswith(forbidden + "/"):
  ```
- **补充**：`FORBIDDEN_PATHS` 缺少 `C:\\Windows`、`C:\\Program Files` 等关键路径；`list_files()` 无安全检查

### P2-4：health_monitor Linux 假设

- **文件**：`health_monitor/implementation.py` L50
- **问题**：`psutil.disk_usage("/")` 在 Windows 上无效
- **修复**：`psutil.disk_usage(os.path.splitdrive(os.getcwd())[0] + "\\")` 或直接用 `psutil.disk_partitions()`

### P2-5：auto_developer 跨技能直接 import

- **文件**：`auto_developer/implementation.py` L102/109
- **问题**：`from skills.agency_orchestrator.implementation import SkillHandler as L1` — 打破技能隔离
- **修复**：改为通过 `skill_handler.execute("agency_orchestrator", "list_roles", {...})` 调用，或使用 `_mcp_call` 模式

### P2-6：agency_orchestrator 路径穿越

- **文件**：`agency_orchestrator/implementation.py` L115
- **问题**：`ROLES_DIR / category / f"{name}.md"` — `category` 和 `name` 未过滤 `..`
- **修复**：添加路径安全校验
  ```python
  if ".." in category or ".." in name:
      return {"success": False, "data": None, "error": "非法路径"}
  ```

### P2-7：agent_loop.py 重复 import + VectorMemory 每次新建

- **文件**：`agent_loop.py` L69/78, L99
- **问题**：`_ensure_mcp_initialized` 重复 `import logging`；`_load_memory` 每次创建新 ChromaDB 连接
- **修复**：删除重复 import；VectorMemory 实例缓存到 `self._vector_memory`

### P2-8：MCPClient 重复方法

- **文件**：`mcp_client.py` L318 `close_all()` 和 L351 `disconnect_all()` 功能重复
- **修复**：保留 `close_all()`，删除 `disconnect_all()`，更新所有引用

## 七、关键架构改进建议

### 7.1 统一路径管理

```python
# core/path_manager.py（新增）
class PathManager:
    PROJECT_ROOT = Path(__file__).parent.parent.resolve()
    
    @classmethod
    def data_dir(cls) -> Path:
        return cls.PROJECT_ROOT / "data"
    
    @classmethod
    def db_path(cls) -> Path:
        return cls.data_dir() / "plector.db"
    
    @classmethod
    def vector_db_path(cls) -> Path:
        return cls.data_dir() / "vector_memory"
```

### 7.2 统一 SSRF 防护

将 `image_handler.py` 中的 SSRF 检测逻辑抽取为 `core/ssrf_guard.py`，供 `web_search`、`image_handler` 等模块复用。

### 7.3 技能沙箱

为 `skill_handler.py` 的动态加载添加路径白名单：
```python
ALLOWED_SKILL_DIRS = {PROJECT_ROOT / "skills", PROJECT_ROOT / "external-skills"}
```
