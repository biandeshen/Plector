# Plector Technical Specification

*Version: 1.0.0*
*Updated: 2026-04-04*

> 本文档定义 Plector 所有技术接口的格式规范。
> 对齐业内标准：OpenAI Function Calling、JSON Schema、MCP、A2A。
> 所有模块实现必须遵循本规范。

---

## 一、Tool/Function Calling 格式

### 1.1 对齐标准

- **OpenAI Function Calling**：事实标准
- **Anthropic Tool Use**：兼容
- **MCP Tools**：预留兼容

### 1.2 Tool Schema 格式

所有工具（包括技能）注册时必须生成以下格式：

```json
{
  "type": "function",
  "function": {
    "name": "skill_name.method_name",
    "description": "一句话描述工具的功能",
    "parameters": {
      "type": "object",
      "properties": {
        "param_name": {
          "type": "string",
          "description": "参数描述"
        }
      },
      "required": ["param_name"],
      "additionalProperties": false
    },
    "strict": true
  }
}
```

### 1.3 字段要求

| 字段 | 必需 | 说明 |
|------|------|------|
| `type` | ✅ | 固定为 `"function"` |
| `function.name` | ✅ | 格式：`{skill_name}.{method_name}` 或 `{tool_name}` |
| `function.description` | ✅ | 一句话描述，供 LLM 判断是否调用 |
| `function.parameters` | ✅ | 完整 JSON Schema（见第二章） |
| `function.parameters.type` | ✅ | 固定为 `"object"` |
| `function.parameters.properties` | ✅ | 参数定义 |
| `function.parameters.required` | ✅ | 必需参数列表 |
| `function.parameters.additionalProperties` | ✅ | 固定为 `false` |
| `function.strict` | ⚠️ | OpenAI strict mode，建议 `true` |

### 1.4 Tool Call 格式（LLM 返回）

```json
{
  "id": "call_abc123",
  "type": "function",
  "function": {
    "name": "health_monitor.check_health",
    "arguments": "{}"
  }
}
```

### 1.5 Tool Result 格式（返回给 LLM）

```json
{
  "success": true,
  "data": {
    "cpu": 12.5,
    "memory": 45.0,
    "disk": 30.0,
    "status": "healthy"
  },
  "error": null
}
```

### 1.6 错误格式（工具调用失败）

```json
{
  "success": false,
  "data": null,
  "error": "技能 health_monitor 不存在"
}
```

## 二、JSON Schema 规范

### 2.1 对齐标准

- **JSON Schema Draft 2020-12**：最新版
- **OpenAI Function Calling**：子集

### 2.2 支持的类型

| JSON Schema 类型 | Python 类型 | 示例 |
|-----------------|-------------|------|
| `string` | `str` | `"hello"` |
| `integer` | `int` | `42` |
| `number` | `float` | `3.14` |
| `boolean` | `bool` | `true` |
| `array` | `list` | `[1, 2, 3]` |
| `object` | `dict` | `{"key": "value"}` |
| `null` | `None` | `null` |

### 2.3 参数定义格式

```json
{
  "param_name": {
    "type": "string",
    "description": "参数的用途描述",
    "default": "默认值（可选）",
    "enum": ["选项1", "选项2"]（可选）
  }
}
```

### 2.4 嵌套对象

```json
{
  "type": "object",
  "properties": {
    "user": {
      "type": "object",
      "properties": {
        "name": {"type": "string", "description": "用户名"},
        "age": {"type": "integer", "description": "年龄"}
      },
      "required": ["name"],
      "additionalProperties": false
    }
  },
  "required": ["user"],
  "additionalProperties": false
}
```

### 2.5 数组

```json
{
  "type": "array",
  "items": {
    "type": "string",
    "description": "文件路径"
  },
  "description": "文件路径列表"
}
```

### 2.6 自动生成规则

从 Python 函数签名自动生成 JSON Schema 时：

```python
def example(name: str, count: int = 5, options: list = None) -> str:
    """
    示例函数

    参数:
        name: 用户名称
        count: 数量，默认 5
        options: 选项列表

    返回:
        结果字符串
    """
    ...
```

自动生成：

```json
{
  "type": "object",
  "properties": {
    "name": {
      "type": "string",
      "description": "用户名称"
    },
    "count": {
      "type": "integer",
      "description": "数量，默认 5",
      "default": 5
    },
    "options": {
      "type": "array",
      "items": {"type": "string"},
      "description": "选项列表"
    }
  },
  "required": ["name"],
  "additionalProperties": false
}
```

## 三、skill.json 规范

### 3.1 完整格式

```json
{
  "name": "health_monitor",
  "description": "获取系统健康状态，包括 CPU、内存、磁盘使用率",
  "version": "1.0.0",
  "tier": "tier_1_system",
  "dependencies": [],
  "events_produced": ["health.degraded", "health.recovered"],
  "events_consumed": [],
  "methods": {
    "check_health": {
      "description": "执行健康检查，无需参数",
      "params": {
        "type": "object",
        "properties": {},
        "required": [],
        "additionalProperties": false
      },
      "returns": {
        "type": "object",
        "properties": {
          "cpu": {"type": "number", "description": "CPU 使用率（百分比）"},
          "memory": {"type": "number", "description": "内存使用率（百分比）"},
          "disk": {"type": "number", "description": "磁盘使用率（百分比）"},
          "status": {"type": "string", "description": "健康状态：healthy 或 degraded"}
        },
        "required": ["cpu", "memory", "disk", "status"],
        "additionalProperties": false
      }
    }
  }
}
```

### 3.2 字段要求

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `name` | string | ✅ | 技能名称，唯一，与目录名一致 |
| `description` | string | ✅ | 简短描述，供 LLM 和人类阅读 |
| `version` | string | ✅ | 语义化版本号 `x.y.z` |
| `tier` | string | ✅ | `tier_0_kernel` / `tier_1_system` / `tier_2_functional` / `tier_3_tool` |
| `dependencies` | array | ✅ | 依赖的其他技能名称列表 |
| `events_produced` | array | ✅ | 本技能发布的事件列表 |
| `events_consumed` | array | ✅ | 本技能订阅的事件列表 |
| `methods` | object | ✅ | 对外暴露的方法签名 |

### 3.3 methods 字段要求

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `methods.<name>.description` | string | ✅ | 方法描述 |
| `methods.<name>.params` | object | ✅ | 完整 JSON Schema |
| `methods.<name>.params.type` | string | ✅ | 固定为 `"object"` |
| `methods.<name>.params.properties` | object | ✅ | 参数定义 |
| `methods.<name>.params.required` | array | ✅ | 必需参数列表 |
| `methods.<name>.params.additionalProperties` | boolean | ✅ | 固定为 `false` |
| `methods.<name>.returns` | object | ⚠️ | 返回值 JSON Schema（推荐） |

### 3.4 与 MCP Tools 的对应

| Plector skill.json | MCP Tool |
|-------------------|----------|
| `name` | `name` |
| `description` | `description` |
| `methods.<name>.params` | `inputSchema` |
| `methods.<name>.returns` | 无（MCP 不定义输出） |

## 四、事件格式规范

### 4.1 事件命名

格式：`<domain>.<action>`

规则：
1. 全部小写
2. 点号分隔领域和动作
3. 动作用过去式

### 4.2 领域列表

| 领域 | 用途 | 示例 |
|------|------|------|
| health | 系统健康 | `health.degraded`, `health.recovered` |
| error | 错误处理 | `error.classified`, `error.stored` |
| skill | 技能管理 | `skill.failed`, `skill.eliminate.proposal` |
| test | 测试相关 | `test.failed`, `test.passed` |
| closure | 闭环执行 | `closure.completed`, `closure.failed` |
| code | 代码相关 | `code.written`, `code.analyzed` |
| agent | Agent 相关 | `agent.started`, `agent.completed` |

### 4.3 事件负载格式

```json
{
  "event_type": "health.degraded",
  "timestamp": "2026-04-04T10:00:00Z",
  "source": "health_monitor",
  "data": {
    "cpu": 85.0,
    "memory": 90.0,
    "disk": 45.0
  }
}
```

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `event_type` | string | ✅ | 事件类型 |
| `timestamp` | string | ✅ | ISO 8601 时间戳 |
| `source` | string | ⚠️ | 发布者名称 |
| `data` | object | ✅ | 事件数据 |

### 4.4 与 A2A 的对应

| Plector 事件 | A2A 概念 |
|-------------|---------|
| 事件负载 | Message |
| `data` | Message.parts |
| `event_type` | Message.role（部分） |

## 五、错误格式规范

### 5.1 工具/技能调用错误

```json
{
  "success": false,
  "data": null,
  "error": "技能 health_monitor 不存在"
}
```

### 5.2 闭环执行错误

```json
{
  "success": false,
  "loop_id": "health_recovery",
  "failed_node": "repair",
  "error": "技能 maintenance 不存在",
  "context": {
    "payload": {"cpu": 85.0}
  }
}
```

### 5.3 错误分类

| 类别 | 触发条件 | 处理方式 |
|------|---------|---------|
| `skill_not_found` | 技能不存在 | 返回错误，Agent Loop 继续 |
| `method_not_found` | 方法不存在 | 返回错误，Agent Loop 继续 |
| `parameter_error` | 参数错误 | 返回错误，LLM 可调整 |
| `execution_error` | 执行异常 | 记录日志，发布 `skill.failed` 事件 |
| `timeout` | 超时 | 记录日志，发布 `skill.failed` 事件 |

### 5.4 与 MCP 的对应

| Plector 错误 | MCP Error |
|-------------|-----------|
| `error` 字段 | `error.message` |
| 错误分类 | `error.code` |

## 七、MCP 兼容层（预留）

### 7.1 目标

让 Plector 的技能可以通过 MCP 协议暴露给外部 LLM。

### 7.2 对应关系

```
Plector                    MCP
─────────────────────────────────────
skill.json                 Tool definition
methods.<name>.params      inputSchema
ToolRegistry.register()    tools/list
ToolRegistry.execute()     tools/call
EventBus                   notifications
```

### 7.3 预留接口

```python
# future: core/mcp_server.py
class MCPServer:
    """将 Plector 技能暴露为 MCP Server"""

    def __init__(self, skill_registry: SkillRegistry):
        self.registry = skill_registry

    def list_tools(self) -> list:
        """返回 MCP 格式的工具列表"""
        tools = []
        for name, info in self.registry.skills.items():
            for method_name, method_info in info["meta"].get("methods", {}).items():
                tools.append({
                    "name": f"{name}.{method_name}",
                    "description": method_info["description"],
                    "inputSchema": method_info["params"],
                })
        return tools

    async def call_tool(self, tool_name: str, arguments: dict) -> dict:
        """执行工具调用"""
        skill_name, method_name = tool_name.split(".")
        result = await self.skill_handler.execute(skill_name, method_name, arguments)
        return {"content": [{"type": "text", "text": json.dumps(result)}]}
```

# future: core/a2a_server.py
class A2AServer:
    """A2A 协议兼容层"""

    def __init__(self, agent_loop: AgentLoop):
        self.agent_loop = agent_loop

    def get_agent_card(self) -> dict:
        """返回 Agent Card"""
        return {
            "name": "Plector",
            "description": "事件驱动的 AI Agent 引擎",
            "skills": [
                {"id": name, "description": info["meta"]["description"]}
                for name, info in self.agent_loop.skill_registry.skills.items()
            ]
        }

    async def handle_task(self, task: dict) -> dict:
        """处理 A2A Task"""
        message = task.get("message", {})
        user_input = message.get("parts", [{}])[0].get("text", "")
        result = await self.agent_loop.run(user_input)
        return {
            "artifacts": [{"parts": [{"type": "text", "text": result}]}]
        }
```

## 九、Python 类型到 JSON Schema 映射

| Python 类型 | JSON Schema | 说明 |
|-------------|-------------|------|
| `str` | `{"type": "string"}` | |
| `int` | `{"type": "integer"}` | |
| `float` | `{"type": "number"}` | |
| `bool` | `{"type": "boolean"}` | |
| `list` | `{"type": "array", "items": {}}` | items 需指定 |

## 十、验证清单

### 10.1 Tool Schema 验证

```python
# scripts/validate_tool_schema.py
import json
import sys

REQUIRED_TOOL_FIELDS = ["type", "function"]
REQUIRED_FUNCTION_FIELDS = ["name", "description", "parameters"]
REQUIRED_PARAMS_FIELDS = ["type", "properties", "required", "additionalProperties"]

def validate(schema):
    errors = []
    for field in REQUIRED_TOOL_FIELDS:
        if field not in schema:
            errors.append(f"Missing '{field}' in tool schema")
    func = schema.get("function", {})
    for field in REQUIRED_FUNCTION_FIELDS:
        if field not in func:
            errors.append(f"Missing 'function.{field}' in tool schema")
    params = func.get("parameters", {})
    for field in REQUIRED_PARAMS_FIELDS:
        if field not in params:
            errors.append(f"Missing 'function.parameters.{field}' in tool schema")
    return errors
```

### 10.2 skill.json 验证

```python
# scripts/validate_skills.py（增强版）
REQUIRED_FIELDS = [
    "name", "description", "version", "tier",
    "dependencies", "events_produced", "events_consumed", "methods"
]

REQUIRED_METHOD_FIELDS = ["description", "params"]
REQUIRED_PARAMS_FIELDS = ["type", "properties", "required", "additionalProperties"]

def validate_skill(data):
    errors = []
    for field in REQUIRED_FIELDS:
        if field not in data:
            errors.append(f"Missing '{field}'")
    for method_name, method in data.get("methods", {}).items():
        for field in REQUIRED_METHOD_FIELDS:
            if field not in method:
                errors.append(f"Missing 'methods.{method_name}.{field}'")
        params = method.get("params", {})
        for field in REQUIRED_PARAMS_FIELDS:
            if field not in params:
                errors.append(f"Missing 'methods.{method_name}.params.{field}'")
    return errors
```
