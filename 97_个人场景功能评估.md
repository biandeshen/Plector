# 97_个人场景功能评估

> 日期: 2026-04-19
> 版本: v1.0
> 状态: 已完成
> 适用场景: 个人 agent 助手（非多用户/团队协作）

---

## 核心结论

Plector 系统功能丰富，但对**个人 agent 助手**场景，以下功能非必需，可以暂时禁用或忽略。

---

## 可延后/禁用的功能

### 1. agency_orchestrator - 多智能体协作

| 项目 | 说明 |
|------|------|
| **定位** | 多角色 YAML 工作流编排 |
| **复杂度** | 174 个 AI 角色、DAG 并行执行、变量传递 |
| **依赖** | Node.js (`servers/agency-orchestrator/dist/cli.js`) |
| **配置** | `config.yaml` 中 30+ 行配置 |
| **问题** | 启动延迟、配置复杂、个人场景用不上 |
| **建议** | 在 `config.yaml` 中设置 `enabled: false` |

### 2. auto_developer - 自动开发流水线

| 项目 | 说明 |
|------|------|
| **定位** | 一键自动开发流水线 |
| **场景** | 面向团队多角色协作 |
| **问题** | 过度复杂，个人场景不需要"团队" |
| **建议** | 视为可选，非必需 |

### 3. self_improver - 系统自我升级

| 项目 | 说明 |
|------|------|
| **定位** | meta 层面自我改进 |
| **风险** | 自我修改代码可能导致不稳定 |
| **价值** | 个人场景下价值有限 |
| **建议** | 暂时禁用 |

### 4. file_utils - 文件操作工具

| 项目 | 说明 |
|------|------|
| **功能** | list_files, copy_file, move_file, delete_file, read_file |
| **问题** | 与 `config.yaml` 中已启用的 `filesystem` MCP Server 功能完全重复 |
| **建议** | 禁用 file_utils skill，使用 MCP filesystem |

---

## 推荐配置

### 保留的 Skills

| Skill | 用途 | 优先级 |
|-------|------|--------|
| `context_refresher` | 防止长对话遗忘 | P0 |
| `memory` | 记忆系统 | P0 |
| `health_monitor` | 健康检查 | P1 |
| `code_writer` | 代码编写 | P1 |
| `test_runner` | 测试运行 | P1 |
| `web_search` | 网络搜索 | P1 |
| `error_knowledge` | 错误记录 | P2 |

### 禁用的 Skills

```yaml
# config.yaml 建议
agency-orchestrator:
  enabled: false

auto_developer:
  enabled: false

self_improver:
  enabled: false
```

### 禁用的 MCP Servers（如有重复）

```yaml
# 保留 filesystem MCP，禁用重复的 file_utils
file_utils:
  enabled: false  # 使用 MCP filesystem 替代
```


## 架构健康度（精简后）

精简后系统更聚焦：

```
✅ 核心稳定: AgentLoop + LLM Client + Skills
✅ 记忆完善: VectorMemory + 艾宾浩斯遗忘曲线
✅ 前端可用: Vue 3 SPA
✅ 安全可靠: SSRF/路径穿越/命令注入防护

❌ 减少: 6 个 worktree 重构 → 聚焦核心
❌ 减少: 10+ skills → 保留 7 个核心
❌ 减少: 多智能体 → 单 agent
```
