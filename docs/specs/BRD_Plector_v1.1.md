## 1. 产品背景与市场机会

### 1.1 市场痛点

当前 AI Agent 框架呈现两极分化：

- **功能全面的框架**（如 LangGraph、AutoGen、CrewAI）提供了强大的编排能力，但代码臃肿、学习曲线陡峭，且缺乏长期运行的自我维护机制。技能数量膨胀后，系统难以治理。
- **极简框架**（如 NanoBot）代码精简、易于上手，但牺牲了自我进化、闭环自愈和技能治理能力，无法满足企业级长期运行需求。

企业需要一种 **既能保持极简内核，又能提供自我进化能力的 Agent 引擎**。

### 1.2 行业趋势

- Gartner 预测，到 2028 年，75% 的企业软件工程将使用 AI Agent 辅助开发。
- 开源 Agent 生态快速发展，但成熟度参差不齐，企业亟需一个 **可落地、可维护、可进化** 的基础设施。
- 本地化部署、数据隐私成为企业级 Agent 的核心诉求。

### 1.3 产品定位

Plector 是一个 **轻量级、可嵌入、具备自我进化能力的 Python Agent 引擎**。它借鉴极简框架的工程实践，同时保留了企业级所需的 **闭环系统、技能治理、事件驱动和自我进化机制**。

**一句话定位**：让 AI Agent 在长期运行中自我维护、自我修复、自我优化。

## 3. 产品核心功能（BRD 层面）

### 3.1 极简内核

- **自主决策循环**：Agent 能够自主推理、调用工具、观察结果并迭代。
- **稳定工具调用**：支持将任意 Python 函数暴露为可调用工具，并保证调用格式的可靠性。
- **事件驱动架构**：组件之间通过异步事件解耦，支持技能协作和闭环触发。
- **动态上下文**：根据项目配置文件自动构建系统提示词，实现人格化和项目记忆。

### 3.2 自我进化闭环

- **自愈流程**：系统能够定义条件触发的自动修复流程（例如测试失败后自动诊断、修复、重测）。
- **技能治理**：对核心能力进行健康度评估、依赖检测和自动淘汰，防止技能膨胀。
- **自我审视**：定期扫描系统状态，生成依赖关系、事件完整性和健康报告。
- **经验学习**：从成功或失败的调用中提取模式，优化后续决策。

### 3.3 技能与工具分层

| 类型 | 定义 | 示例 | 治理 |
|------|------|------|------|
| **核心技能** | 有状态、有事件、有依赖 | 健康监控、错误知识化、自学习 | 完整治理（健康分、淘汰） |
| **工具函数** | 无状态、无事件、无依赖 | 格式转换、网络搜索 | 无治理，仅注册 |

核心技能数量严格控制在 10-15 个，其余能力通过轻量工具函数扩展。

### 3.4 多渠道接入

- **命令行界面**：支持交互式对话和单次查询。
- **WebSocket**（可选）：实时双向通信，用于 Web 界面。
- **HTTP API**（远期）：供外部系统调用。

## 4. 差异化优势

| 对比维度 | LangGraph / CrewAI | NanoBot | **Plector** |
|----------|--------------------|---------|--------------|
| 代码量 | 数万行 | ~4000 行 | **< 5000 行** |
| 自我进化（闭环） | ❌ | ❌ | ✅ |
| 技能治理 | ❌ | ❌ | ✅ |
| 事件驱动 | ❌ | ❌ | ✅ |
| 自主决策 + 工具调用 | 部分 | ✅ | ✅ |
| 可嵌入 | 复杂 | ✅ | ✅ |
| 企业级长期运行 | 需大量二次开发 | 不支持 | **原生支持** |

**核心差异化**：Plector 是唯一在极简内核中内建自我进化能力的 Agent 引擎。

### 7.3 未来演进方向

基于与 Hermes Agent、DeerFlow、OpenClaw 等开源项目的深度对比研究，Plector 未来将向以下方向演进：

| 演进方向 | 目标 | 优先级 |
|----------|------|--------|
| **中间件架构** | 引入中间件链，支持 ThreadData、Memory、Summarization 等中间件 | P0 |
| **增强记忆系统** | 引入 FTS5 全文搜索 + LLM 自动记忆提取 + 上下文压缩 | P0 |
| **多智能体支持** | 工作区隔离 + 子代理池 + 并行执行 | P1 |
| **渠道网关** | 抽象多渠道适配器（WebSocket、Telegram、Discord、钉钉/飞书） | P2 |
| **安全与审计** | RBAC/ABAC + 沙箱执行 + 审计日志 | P0 |

**技术参考**：
- Hermes Agent：多终端后端（Docker/SSH）、FTS5 全文搜索
- DeerFlow：中间件链（9 个中间件）、子代理池、上下文压缩
- OpenClaw：多智能体隔离、优先级路由、斜杠命令系统

### 10.3 集成方案对比

| 方案 | 描述 | 复杂度 | 控制度 |
|------|------|--------|--------|
