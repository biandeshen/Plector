### 明确不在 v1.x 范围内

| 功能 | 原因 |
|------|------|
| 自建前端 Dashboard | 已被 Lobe Chat Provider 替代 |
| 渠道网关（Telegram/钉钉/飞书） | 个人使用不需要多渠道 |
| 技能市场 / 开源社区建设 | 暂不开源，无社区需求 |
| 艾宾浩斯遗忘曲线 / 联想式记忆网络 | Mem0 内部已覆盖，不需要自己实现 |
| FTS5 全文索引 | Mem0 内置 BM25 关键词检索 |
| LangGraph 状态机集成 | 现有 closure_engine 已满足需求 |
| Dialect 编程语言 | 无此需求，YAML 声明式配置已足够 |
| RBAC/ABAC 权限系统 | 单人使用，不需要权限管理 |

## 3. 核心用户故事

### 故事1：助手记住我的偏好
> 作为用户，我告诉助手"我喜欢用 Python 写脚本，偏好 dark mode"，三天后我问"帮我写个自动化脚本"，助手自动用 Python 并输出暗色主题的代码。

**验收标准**：
- Mem0 自动提取并存储偏好
- 新会话中检索到相关记忆并注入上下文
- Agent 回复体现记忆内容

### 故事2：助手有稳定性格
> 作为用户，我编辑 SOUL.md 定义助理性格为"严谨但幽默"，Agent 在所有对话中保持一致风格。

**验收标准**：
- SOUL.md 修改后热加载生效
- 新会话中 Agent 回复风格与 SOUL.md 定义一致

### 故事3：助手在我睡觉时修 bug
> 作为用户，我启动闭环监控，设置 `test.failed` 触发自动修复。凌晨测试失败，Agent 自动诊断、修复、重测，早上我看到修复报告。

**验收标准**：
- 闭环引擎正确触发
- 修复成功率 ≥ 70%
- 生成结构化修复报告

### 故事4：助手能调度专家角色
> 作为用户，我输入"帮我审查这段代码的安全性和性能"，Agent 自动使用 agency-orchestrator 调度安全工程师 + 性能工程师并行审查。

**验收标准**：
- 多角色并行执行
- 输出汇总报告

### 故事5：通过 Lobe Chat 使用助手
> 作为用户，我打开 Lobe Chat，选择"Plector Agent"作为模型，像使用 ChatGPT 一样和我的助手对话，还能看到工具调用的可视化卡片。

**验收标准**：
- Lobe Chat Provider 插件正常运行
- 工具调用卡片正确显示
- 流式响应无卡顿

## 4. 关键数据模型

### 4.1 灵魂文件结构

```
plector_data/
├── soul.md              # AI 人格定义（用户编辑，AI 可建议）
├── user.md              # 用户画像（用户编辑，AI 可建议）
├── agents.md            # 安全规则与工具权限（用户编辑）
├── memory.md            # 长期知识沉淀（AI + 用户共同维护）
└── memory/              # Mem0 持久化存储
    └── ...
```

### 4.2 soul.md 模板

```markdown
# Plector 人格定义

## 姓名
Plector

## 性格
- 严谨：用代码和逻辑说话，不猜测
- 幽默：适当的时候可以开玩笑
- 好奇：对新技术保持开放心态

## 沟通风格
- 简洁直接，不废话
- 用 Markdown 格式回复
- 代码块标注语言类型

## 能力边界
- 不代替用户做不可逆的操作（rm -rf、drop table）
- 不确定时主动询问而非猜测
```

### 4.3 user.md 模板

```markdown
# 用户画像

## 基本信息
- 称呼：[你的名字]
- 职业：全栈开发者
- 技术栈：Python, TypeScript, Vue 3, Docker

## 偏好
- 编辑器：VS Code
- 代码风格：简洁、函数式
- 注释语言：中文
- 操作系统：Windows / WSL2

## 习惯
- 工作时间：上午 9 点到凌晨 1 点
- 喜欢在项目根目录下放 AGENTS.md
- 偏好 YAML 配置而非 JSON
```

## 5. 发布计划

| 阶段 | 时间 | 核心任务 | 产出 |
|------|------|----------|------|
| **Phase 1：人格觉醒** | 第 1-2 周 | 灵魂文件系统落地 | SOUL.md / USER.md 热加载，Agent 首次拥有稳定人格 |
| **Phase 2：记忆觉醒** | 第 3-4 周 | Mem0 记忆中间件集成 | Mem0MemoryMiddleware 完成，Agent 开始自动记忆 |
| **Phase 3：颜值觉醒** | 第 5-6 周 | Lobe Chat Provider 插件 | 在 Lobe Chat 里使用 Plector，告别纯 CLI |

## 6. 非功能需求

| 类别 | 要求 |
|------|------|
| **性能** | Agent 循环 < 100ms，事件分发 < 10ms |
| **可靠性** | 闭环修复成功率 ≥ 70% |
| **安全性** | 工具沙箱 + 用户确认不可逆操作 |
| **可维护性** | 核心代码 < 5000 行，核心技能 ≤ 15 个 |
| **易用性** | 安装后 5 分钟内可运行第一个示例 |

## 7. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| Mem0 学习成本 | 中 | Mem0 API 简洁，接入代码不超过 50 行 |
| Lobe Chat 协议变更 | 中 | Provider 基于标准 WebSocket 协议，低耦合 |
| 闭环修复误判导致事故 | 高 | 保留人工通知 + 回滚能力 |
| 记忆数据膨胀 | 中 | Mem0 内置衰减机制 |
