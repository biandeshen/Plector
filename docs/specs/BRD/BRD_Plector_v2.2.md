# Plector 商业需求文档（BRD）v2.2

**版本**: 2.2
**状态**: 已定稿
**日期**: 2026-04-28

## 1. 产品定位

**一句话定位**：Plector 是个人开发者的超级助理引擎——有性格、有记忆、能自愈、能联动。

**核心价值主张**：
- **有性格**：灵魂文件系统（SOUL.md / USER.md / MEMORY.md），Agent 拥有稳定人格和成长轨迹
- **有记忆**：Mem0 驱动的智能记忆层，接近真人的记忆体验
- **能自愈**：闭环引擎自动检测、诊断、修复故障，修复成功率 ≥ 70%
- **能联动**：MCP 原生集成 + Lobe Chat Provider 前端 + 多智能体工作流

## 2. 目标用户

**唯一目标用户**：你（个人开发者 / Vibe Coder）

你的核心需求：
1. 一个越来越懂你的助手（记忆要极强）
2. 一个能在你睡觉时帮你修 bug 的助手（闭环自愈）
3. 一个有稳定性格的助手（灵魂文件系统）
4. 一个能调度多种能力协作的助手（多智能体 + MCP 生态）

## 4. 核心差异化

- **灵魂文件系统**：Agent 拥有可读可写的性格文件，人机共同维护
- **Mem0 记忆层**：语义 + BM25关键词 + 实体匹配多信号融合，p95 检索延迟 0.2s
- **闭环自愈引擎**：事件驱动的自动修复流水线，无需人工值守
- **MCP 原生集成**：零维护成本接入整个 MCP 工具生态

## 5. 成功指标

| 指标 | 目标值 |
|------|--------|
| 闭环修复成功率 | ≥ 70% |
| 核心代码量 | < 5000 行 |
| 核心技能数量 | ≤ 15 个 |
| 安装到首次运行 | < 5 分钟 |

## 6. 商业模式

**个人使用，暂不商业化**。未来如需变现，采用免费增值模式（开源核心 + 云服务增值）。
