## 什么是 Plector？

### 事件驱动的 AI Agent 引擎

Plector 是一个创新的 AI Agent 运行时框架，融合了：
- **自主决策**：ReAct 循环推理
- **技能治理**：可插拔的技能系统
- **闭环自愈**：条件触发与自动修复

---

## 我们解决什么问题？

### AI Agent 落地的三大挑战

| 挑战 | 传统方案 | Plector 方案 |
|------|----------|--------------|
| 工具调用混乱 | 硬编码工具链 | 技能注册与治理 |
| 上下文丢失 | 手动传递状态 | 事件总线解耦 |
| 异常无法自愈 | 人工干预 | 闭环引擎自动恢复 |

## 核心模块：Agent Loop

### 自主决策引擎

```python
class AgentLoop:
    def run(self, user_input: str) -> str:
        # ReAct 循环
        while not done:
            thought = self.reason(messages)      # 思考
            action = self.act(thought)          # 执行
            result = self.observe(action)       # 观察
            messages.append(result)
        return final_answer
```

**关键特性**：
- 统一的工具调用接口
- 迭代次数限制保护
- 会话状态管理

## 核心模块：Event Bus

### 事件驱动架构

```python
# 发布订阅模式
event_bus.subscribe("skill.completed", on_skill_done)
event_bus.subscribe("error.*", on_error)

# 异步事件处理
await event_bus.publish("task.started", {"task_id": "123"})
```

**优势**：
- 模块间完全解耦
- 支持通配符订阅
- 可插拔后端（内存/Redis）
