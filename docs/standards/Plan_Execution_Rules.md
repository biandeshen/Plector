## 一、Plan.md 概述

### 1.1 什么是 Plan.md

`Plan.md` 是复杂任务执行过程中的**执行日志**，记录：
- 假设和验证结果
- 执行的步骤和结果
- 遇到的问题和解决方案
- 后续计划

### 1.2 Plan.md vs PLAN_TEMPLATE.md

| 文档 | 用途 | 创建时机 |
|------|------|----------|
| `PLAN_TEMPLATE.md` | 模板文件（根目录） | 定义模板格式 |
| `Plan.md` | 执行日志（项目根目录） | 任务开始时创建 |

### 1.3 什么时候需要创建 Plan.md

| 场景 | 是否需要 | 说明 |
|------|----------|------|
| 简单单行修改 | ❌ 不需要 | 直接执行 |
| 修改单个文件 | ❌ 不需要 | 除非涉及复杂逻辑 |
| 修改多个文件 | ✅ 需要 | 确保进度可追踪 |
| Bug 修复 | ✅ 需要 | 记录分析过程 |
| 新功能开发 | ✅ 需要 | 记录设计决策 |
| 重构任务 | ✅ 需要 | 记录影响范围 |
| 文档编写 | ❌ 不需要 | 除非涉及多文档 |

## 三、执行日志规范

### 3.1 日志格式

```
HH:MM | [动作] | [结果] | [下一步]
```

### 3.2 日志示例

```
14:30 | 读取 core/event_bus.py | ✅ 成功，理解了事件分发逻辑 | 开始修改
14:32 | 修改 EventBus.emit() 方法 | ✅ 成功 | 运行测试验证
14:35 | 运行 pytest | ❌ 失败，2 个测试未通过 | 分析失败原因
14:38 | 分析失败原因 | ✅ 发现是参数类型不匹配 | 修复类型问题
14:40 | 修复类型问题 | ✅ 成功 | 再次运行测试
14:42 | 运行 pytest | ✅ 3/3 通过 | 准备提交
14:45 | 提交代码 | ✅ 成功 | 任务完成
```

### 3.3 结果标记

| 标记 | 含义 | 使用场景 |
|------|------|----------|
| ✅ | 成功 | 操作正常完成 |
| ❌ | 失败 | 操作未成功，需要分析 |
| ⏸️ | 暂停 | 等待用户确认 |
| 🔄 | 进行中 | 操作未完成，需要继续 |

### 3.4 日志原则

| 原则 | 说明 |
|------|------|
| **时间顺序** | 按时间顺序记录，最新的在最下方 |
| **原子操作** | 每条日志记录一个原子操作 |
| **准确描述** | 动作描述清晰，结果准确 |
| **即时更新** | 操作完成后立即记录 |

## 四、假设验证记录

### 4.1 假设记录格式

```markdown
### 假设 N
- **假设内容**：[描述]
- **假设依据**：[为什么这么认为]
- **验证时间**：HH:MM
- **验证结果**：✅ 通过 / ❌ 否定
- **后续处理**：[通过后执行什么 / 否定后如何处理]
```

### 4.2 假设示例

```markdown
### 假设 1
- **假设内容**：EventBus.emit() 需要支持异步事件分发
- **假设依据**：handle_event() 是 async 函数，emit() 必须是 async
- **验证时间**：14:30
- **验证结果**：✅ 通过
- **后续处理**：将 emit() 改为 async def emit()
```

### 4.3 假设否定处理

```markdown
### 假设 2
- **假设内容**：可以直接修改 EventBus 类，不需要创建子类
- **假设依据**：现有代码结构简单
- **验证时间**：14:35
- **验证结果**：❌ 否定
- **后续处理**：停止当前方案，重新分析。原有代码有多个子类继承，需要创建子类覆盖方法。
```

## 五、问题记录

### 5.1 问题记录格式

```markdown
### 问题 N
- **问题描述**：...
- **发现时间**：HH:MM
- **分析**：...
- **解决方案**：...
- **状态**：待解决 / 已解决
- **解决时间**：HH:MM
```

### 5.2 问题示例

```markdown
### 问题 1
- **问题描述**：pytest 报告 `test_event_handler` 失败
- **发现时间**：14:35
- **分析**：
  - 期望：事件被同步分发
  - 实际：事件被异步分发
  - 原因：修改 emit() 为 async 后，事件分发变为异步
- **解决方案**：修改测试用例，添加 `await` 等待
- **状态**：已解决
- **解决时间**：14:40
```

## 六、模板使用

### 6.1 使用 PLAN_TEMPLATE.md

创建 Plan.md 时，可以参考根目录的 `PLAN_TEMPLATE.md`：

```bash
# 复制模板创建 Plan.md
cp PLAN_TEMPLATE.md Plan.md

# 或者直接创建
cat > Plan.md << 'EOF'
# [任务名称]

## 执行日志

| 时间 | 动作 | 结果 | 下一步 |
|------|------|------|--------|
| HH:MM | ... | ... | ... |

## 结论
EOF
```

### 6.2 任务结束后清理

任务完成后，可以选择：
1. **保留 Plan.md**：作为执行记录，便于回顾
2. **删除 Plan.md**：如果任务简单，不需要保留
