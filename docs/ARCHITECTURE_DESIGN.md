# 文档架构设计：工具规范与项目规范分离

> 版本：v1.0.0 | 最后更新：2026-04-28

---

## 一、设计目标

### 1.1 问题背景

传统的文档系统中，通用工具规范（如假设验证、提交规范、代码规范）与项目专用规范混在一起，导致：
- **重复**：同一规范在多个项目中复制粘贴
- **维护困难**：修改一处需要同步多处
- **职责不清**：无法区分"工具使用规范"和"项目业务规范"

### 1.2 设计目标

```
┌─────────────────────────────────────────────────────────────┐
│  核心目标：工具规范与项目规范分离                              │
│                                                              │
│  - Claude Code 工具规范 → 统一仓库，跨项目复用                │
│  - Plector 项目规范 → 项目独有，独立维护                       │
│  - 双索引架构 → 各引用各自的 DOCS_INDEX.md                    │
└─────────────────────────────────────────────────────────────┘
```

---

## 二、架构设计

### 2.1 双仓库结构

```
┌─────────────────────────────────────────────────────────────┐
│  E:/笔记/ (Git 仓库)                                        │
│  ├── Claude Code规范/                                        │
│  │   ├── DOCS_INDEX.md         ← Claude Code 工具规范索引    │
│  │   ├── Agent_Behavior_Rules.md                              │
│  │   ├── Coding_Convention.md                                  │
│  │   ├── Commit_Convention.md                                  │
│  │   └── ...                                                   │
│  └── 其他笔记/                                               │
└─────────────────────────────────────────────────────────────┘
                              ↓ 引用
┌─────────────────────────────────────────────────────────────┐
│  e:/产品/Plector/ (Git 仓库)                                 │
│  ├── CLAUDE.md                 ← 项目入口，引用双索引         │
│  ├── SOUL.md                   ← Plector 灵魂                 │
│  ├── docs/                                                      │
│  │   ├── DOCS_INDEX.md         ← Plector 项目文档索引         │
│  │   ├── PLECTOR_SKILLS.md                                    │
│  │   ├── specs/                                                │
│  │   ├── guides/                                              │
│  │   └── api/                                                 │
│  └── skills/                                                   │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 索引职责划分

| 索引 | 位置 | 职责 | 包含内容 |
|------|------|------|----------|
| **Claude Code 工具规范索引** | `E:/笔记/Claude Code规范/DOCS_INDEX.md` | 工具使用规范 | 行为规则、代码规范、提交规范、语言约定等 |
| **Plector 项目文档索引** | `e:/产品/Plector/docs/DOCS_INDEX.md` | 项目业务文档 | 技能系统、技术规格、规格文档、用户指南等 |

### 2.3 CLAUDE.md 的角色

```markdown
# CLAUDE.md 不再是"规范大全"
# 而是"双索引入口"

## Claude Code 工具规范（引用）
→ E:/笔记/Claude Code规范/DOCS_INDEX.md

## Plector 项目文档（引用）
→ docs/DOCS_INDEX.md
```

## 三、文档归属判定规则

### 3.1 判断流程

```
文档内容 → 判断是否与特定项目绑定？
    │
    ├── 否（通用）→ Claude Code 工具规范
    │               → 放到 E:/笔记/Claude Code规范/
    │
    └── 是（项目专用）→ 项目文档
                        → 放到项目 docs/ 目录
```

### 3.2 归属示例

| 文档 | 判断 | 放置位置 |
|------|------|----------|
| 假设验证规则 | 通用（任何 Agent 都适用） | `E:/笔记/Claude Code规范/Agent_Behavior_Rules.md` |
| 提交规范 | 通用（任何项目都适用） | `E:/笔记/Claude Code规范/Commit_Convention.md` |
| Plector 技能系统 | 项目专用（Plector 特有） | `e:/产品/Plector/docs/PLECTOR_SKILLS.md` |
| Plector 技术规格 | 项目专用（Plector 特有） | `e:/产品/Plector/docs/standards/Technical_Spec_Plector.md` |

## 四、实施规范

### 4.1 新增规范时的流程

```
1. 判断归属（通用/项目专用）
    │
    ├── 通用 → 放到 E:/笔记/Claude Code规范/
    │           → 更新 E:/笔记/Claude Code规范/DOCS_INDEX.md
    │
    └── 项目专用 → 放到项目 docs/ 目录
                   → 更新项目 docs/DOCS_INDEX.md
```

### 4.2 CLAUDE.md 更新规范

```markdown
# CLAUDE.md 更新规则

1. 新增通用规范引用：
   - 直接链接到 E:/笔记/Claude Code规范/DOCS_INDEX.md
   - 或链接到具体规范文件

2. 新增项目规范引用：
   - 直接链接到 docs/DOCS_INDEX.md
   - 或链接到具体项目文档
```

### 4.3 同步机制

| 操作 | 说明 |
|------|------|
| `cd E:/笔记/ && git log -1 --format="%h %s"` | 查看通用规范版本 |
| `cd e:/产品/Plector/ && git log -1 --format="%h %s"` | 查看项目版本 |

## 五、优势

### 5.1 DRY 原则

```
之前：每个项目都复制一份"假设验证规则"
现在：所有项目引用同一份 E:/笔记/Claude Code规范/Agent_Behavior_Rules.md
```

### 5.2 版本一致性

```
通用规范更新 → 所有引用该项目自动获得最新版本
（通过 Git 追踪）
```

### 5.3 职责清晰

```
Claude Code 工具规范 → E:/笔记/
Plector 项目规范 → e:/产品/Plector/
CLAUDE.md → 双索引入口
```
