## 一、现状分析

### 1.1 系统架构概览

```
plector/
├── core/                      # 核心引擎 (11 个模块)
│   ├── agent_loop.py         # ReAct 循环 (10,837 行)
│   ├── event_bus.py          # 事件总线 (2,672 行)
│   ├── skill_registry.py     # 技能注册 (4,268 行)
│   ├── skill_handler.py      # 技能执行 (3,284 行)
│   ├── closure_engine.py     # 闭环引擎 (2,488 行)
│   ├── context_builder.py     # 上下文构建 (2,057 行)
│   ├── function_calling.py    # 函数调用 (2,935 行)
│   ├── llm_client.py          # LLM 客户端 (6,031 行)
│   ├── mcp_client.py          # MCP 客户端 (15,804 行)
│   ├── governance.py          # 技能治理 (2,126 行)
│   └── vector_memory.py       # 向量存储 (8,922 行)
│
├── skills/                    # 核心技能 (8 个)
│   ├── agency_orchestrator/    # 多智能体编排
│   ├── auto_developer/        # 自动开发
│   ├── code_writer/           # 代码编写
│   ├── error_knowledge/        # 错误知识库
│   ├── file_utils/            # 文件操作
│   ├── health_monitor/        # 健康监控
│   ├── memory/                # 记忆管理
│   ├── test_runner/          # 测试运行
│   └── web_search/            # 网页搜索
│
├── servers/                   # MCP Servers
│   ├── filesystem_server.py
│   └── sqlite_server.py
│
├── channels/                  # 接入渠道
│   ├── cli.py
│   └── websocket.py
│
└── config/
    ├── config.yaml
    └── closed_loops.yaml
```

### 1.2 核心指标

| 指标 | 当前值 | 目标值 |
|------|--------|--------|
| 核心代码量 | ~60,000 行 | < 50,000 行 |
| 技能数量 | 8 个 | ≤ 15 个 |
| 函数平均长度 | 部分超标 | ≤ 50 行 |
| 单元测试覆盖 | 未达标 | ≥ 80% |
| 启动时间 | - | < 2s |

## 二、问题识别

### 2.1 架构层面

| 问题 | 影响 | 优先级 |
|------|------|--------|
| `agent_loop.py` 超过 10k 行 | 可维护性差 | P0 |
| `mcp_client.py` 超过 15k 行 | 难以测试和扩展 | P0 |
| `image_handler.py` 超过 20k 行 | 单文件过大 | P1 |
| 缺乏统一的错误处理层 | 系统稳定性 | P0 |
| 配置加载分散 | 维护困难 | P1 |

### 2.2 性能层面

| 问题 | 影响 | 优先级 |
|------|------|--------|
| LLM 调用未实现流式响应 | 用户体验 | P1 |
| 事件总线内存泄漏风险 | 长时间运行稳定性 | P0 |
| 技能冷加载延迟 | 首次调用慢 | P1 |
| 向量检索未优化 | 记忆搜索慢 | P1 |

### 2.3 安全层面

| 问题 | 影响 | 优先级 |
|------|------|--------|
| 敏感信息硬编码风险 | 安全合规 | P0 |
| 技能执行沙箱缺失 | 系统安全 | P0 |
| LLM 输出未过滤 | 内容安全 | P1 |
| MCP 通信无加密 | 数据安全 | P1 |

### 2.4 技术债

| 类型 | 描述 | 工作量 |
|------|------|--------|
| 重构 | `image_handler.py` 拆分 | 3 人日 |
| 重构 | `mcp_client.py` 模块化 | 5 人日 |
| 重构 | 统一错误处理层 | 2 人日 |
| 优化 | 技能热加载机制 | 3 人日 |
| 新增 | 单元测试补全 | 5 人日 |
| 新增 | 性能监控埋点 | 2 人日 |

## 三、升级目标 (v2.0)

### 3.1 功能增强

- [ ] **多会话并发**: 支持 100+ 并发会话
- [ ] **流式响应**: LLM 输出流式渲染
- [ ] **技能热更新**: 无重启更新技能
- [ ] **增强记忆**: 支持结构化记忆查询
- [ ] **审计日志**: 完整操作审计

### 3.2 性能提升

- [ ] 启动时间 < 2s
- [ ] 技能调用延迟 < 500ms
- [ ] 内存占用降低 30%
- [ ] 支持水平扩展

### 3.3 安全加固

- [ ] 技能执行沙箱
- [ ] 敏感信息加密存储
- [ ] LLM 输出内容过滤
- [ ] 操作审计日志

### 3.4 可观测性

- [ ] 健康检查端点
- [ ] 指标采集 (Prometheus 格式)
- [ ] 链路追踪集成
- [ ] 日志结构化输出
