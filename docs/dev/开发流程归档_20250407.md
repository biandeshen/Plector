## 📋 本次讨论的核心内容

### 1. 升级目标
将 Plector 从 v1.2 升级到 v2.0，主要目标：
- 核心代码量 < 50,000 行
- 技能数量 ≤ 15 个
- 函数平均长度 ≤ 50 行
- 单元测试覆盖 ≥ 80%

### 2. 已确定的技术方案

#### 核心架构
```
plector/
├── core/                      # 核心引擎
│   ├── error_handler.py      # 新增: 统一错误处理
│   ├── event_bus_v2.py       # 优化: 内存管理
│   └── security/             # 新增: 安全模块
│       ├── secrets_manager.py
│       └── output_filter.py
│
├── skills/                    # 核心技能 (当前 8 个)
│
├── servers/                   # MCP Servers
│
├── channels/                  # 接入渠道
│
└── config/
    └── closed_loops.yaml
```

#### 5 阶段实施计划
| 阶段 | 目标 | 时间 | 关键任务 |
|------|------|------|----------|
| Phase 1 | 核心稳定性增强 | 2 周 | 错误处理、内存优化、安全加固 |
| Phase 2 | 性能优化 | 2 周 | 流式响应、热更新、向量化 |
| Phase 3 | 模块化重构 | 3 周 | mcp_client 模块化、agent_loop 精简 |
| Phase 4 | 可观测性 | 1 周 | 健康检查、指标采集、链路追踪 |
| Phase 5 | 新功能扩展 | 2 周 | 按需扩展 |

**总计**: 10 周

### 3. 已识别的 P0 问题

| 问题 | 影响 | 方案 |
|------|------|------|
| `agent_loop.py` 超过 10k 行 | 可维护性差 | 重构至 < 500 行 |
| `mcp_client.py` 超过 15k 行 | 难以测试和扩展 | 模块化拆分 |
| 事件总线内存泄漏 | 长时运行稳定性 | 内存优化 |
| 敏感信息硬编码风险 | 安全合规 | 加密存储 |
| 技能执行沙箱缺失 | 系统安全 | 沙箱基础实现 |
