## 二、核心组件

### 2.1 ClosureEngine (闭环引擎)

**文件**: `core/closure_engine.py`

**配置**: `config/closed_loops.yaml`

**触发机制**: 事件驱动 + YAML 声明式

```python

  trigger_on: ["test.failed"]
  nodes:
    - type: skill

      method: record_error
    - type: skill
      skill: error_knowledge
      method: classify_error
```

### 2.2 EventBus (事件总线)

**文件**: `core/event_bus_v2.py`

**格式**: CloudEvents 1.0

```python
@dataclass
class Event:
    specversion: str = "1.0"
    id: str = ""
    source: str = "plector"
    type: str = ""        # 如 "test.failed"
    time: str = ""
    data: dict = field(default_factory=dict)
```

**发布点**:
| 事件 | 发布位置 |
|------|----------|
| `error.stored` | skills/error_knowledge |
| `health.{status}` | skills/health_monitor |
| `memory.stored` | skills/memory |
| `context.preserved` | drive_plector_core.py |
| `code.written` | skills/code_writer |
| `search.completed` | skills/web_search |

## 三、上下文保鲜机制

**文件**: `skills/context_refresher/implementation.py`

### 保鲜数据结构

```python
@dataclass
class GSDContext:
    session_id: str
    goal: str = ""                    # 初始目标
    constraints: list[str] = []       # 约束条件
    completed: list[str] = []          # 已完成
    in_progress: list[str] = []       # 进行中
    turn_count: int = 0               # 轮次计数
    last_refresh: float = 0.0        # 上次保鲜时间
    goal_version: int = 1             # 目标版本
```

### 触发条件

| 方法 | 触发条件 |
|------|----------|
| `preserve()` | 每 10 轮自动触发 |
| `re_anchor()` | 用户明确修改目标 |

### 保鲜流程

```
preserve():
  1. 从最近20轮对话提取上下文
  2. 提取 goal (首条用户消息)
  3. 提取 constraints (不要/必须等关键词)
  4. 提取 completed/in_progress
  5. 存入 vector_memory.context_saver
  6. 发布 context.preserved 事件
```

## 四、记忆强化机制

**文件**: `core/vector_memory_v2.py`

### 艾宾浩斯遗忘曲线

```python
DECAY_COEFFICIENTS = {
    5: 0.9,   # 5小时间隔 - 高衰减率
    10: 0.8,  # 10小时间隔
    30: 0.5,  # 30小时间隔
    50: 0.3,  # 50小时间隔 - 低衰减率
}

# 衰减公式: intensity = e^(-elapsed_hours / (decay_coef * 10))
```

### 记忆强度等级

| 等级 | 阈值 | 描述 |
|------|------|------|
| ALIVE | >= 0.8 | 活跃记忆 |
| NORMAL | >= 0.5 | 正常记忆 |
| FADING | >= 0.2 | 衰退记忆 |
| FORGOTTEN | < 0.2 | 可清除 |

### 强化机制

```python
reinforce_memory(doc_id):
  intensity = min(1.0, current + 0.2)  # +0.2 增幅
  repetition_count += 1
```

## 七、配置参考

```yaml

error_record_loop:
  trigger_on: ["test.failed"]
  nodes:
```

## 八、优化建议

### 短期 (1-2 小时)

1. **集成 context_refresher 到主循环**
   - 在 AgentLoop 中维护 turn_count
   - 每 10 轮自动调用 preserve

2. **实际执行复杂度分析结果**
   - 将 recommended_actions 传给 LLM 决策
   - 或自动触发 context_refresher

### 中期 (半天)

3. **完善事件发布点**
   - 确保 test.failed 等事件正确发布
   - 连接 ClosureEngine 与主循环

4. **添加记忆强化自动触发**
   - 检索记忆后自动强化
   - 或每轮结束后强化本次对话记忆
