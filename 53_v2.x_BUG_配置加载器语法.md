# 修复配置加载器环境变量语法（Claude Code 可执行版）

> 问题：
>   配置文件使用 ${VAR:-default} 语法
>   config_loader.py 只支持 ${VAR} 格式
>   导致环境变量无法正确解析
>
> 共 1 步，直接替换函数。

---

## 第一步：替换 _resolve_env_vars 函数

修改 `core/config_loader.py`，找到 `_resolve_env_vars` 函数，替换为：

```python
def _resolve_env_vars(obj: Any) -> Any:
    """递归替换配置中的环境变量引用（支持 ${VAR} 和 ${VAR:-default}）"""
    if isinstance(obj, str):
        # 匹配 ${VAR_NAME} 或 ${VAR_NAME:-default} 格式
        pattern = r"\$\{([^}:]+)(?::-([^}]*))?\}"

        def replacer(match):
            var_name = match.group(1)
            default_value = match.group(2) or ""
            value = os.environ.get(var_name, "")
            if not value:
                if default_value:
                    return default_value
                else:
                    logger.warning(f"环境变量未设置: {var_name}")
            return value

        return re.sub(pattern, replacer, obj)

    elif isinstance(obj, dict):
        return {key: _resolve_env_vars(value) for key, value in obj.items()}

    elif isinstance(obj, list):
        return [_resolve_env_vars(item) for item in obj]

    else:
        return obj
```
