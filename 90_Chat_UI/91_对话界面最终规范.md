### 三、工具调用卡片（完整规范）

#### 3.1 卡片结构

每个工具调用是一个独立卡片，包含 4 个部分：

```
┌─────────────────────────────────────────────────────────┐
│ 🔧 web_search                  完成 0.8s          ▼    │  ← 工具头
├─────────────────────────────────────────────────────────┤
│ 参数: {"query": "今日热点新闻", "count": 10}             │  ← 参数
├─────────────────────────────────────────────────────────┤
│ 思考: 用户想要了解今天的热点新闻，我应该搜索最新内容      │  ← 思考（可选）
├─────────────────────────────────────────────────────────┤
│ 结果: 找到 10 条相关新闻...                              │  ← 结果
└─────────────────────────────────────────────────────────┘
```

#### 3.2 各部分详情

**工具头（第一行，始终显示）**：
- 左侧：工具图标 🔧
- 工具名：蓝色（#38bdf8），加粗，如 `web_search`、`read_file`
- 中间：状态标签
  - "执行中"：黄色背景，有旋转动画
  - "完成 0.8s"：绿色背景
  - "错误"：红色背景
- 右侧：展开/折叠箭头 ▼/▲
- hover 时背景变亮

**参数（展开后显示）**：
- 标签："参数:"，灰色小字
- 内容：JSON 格式，等宽字体，深色背景
- 去掉外层 `jsonrpc`/`success`/`data` 外壳，只选）**：
- 标签："思考:"，灰色小字
- 内容：LLM 在调用工具前的思考过程
- 来源：`<think>` 标签内的内容
- 颜色：显示有用字段

**思考（展开后显示，可浅灰色（#94a3b8）

**结果（展开后显示）**：
- 标签："结果:"，灰色小字
- 内容：工具执行结果
- 去掉外层 `jsonrpc`/`result`/`success` 外壳，只显示有用字段
- 如果是 JSON，格式化显示
- 如果是文本，直接显示
- 如果太长，只显示前 500 字符，点击"展开全部"

#### 3.3 折叠/展开行为

- **默认状态**：折叠，只显示工具头
- **点击工具头**：展开/折叠
- **多个工具**：每个独立折叠，互不影响
- **动画**：展开时有滑动动画（200ms）

#### 3.4 统计行

在工具卡片上方，显示统计信息：

```
> 4 个工具调用 | 6/8 步骤
```

- 灰色文字，字号 13px
- 点击可展开/折叠所有工具卡片

### 四、过程信息（Thinking）

#### 4.1 什么是过程信息

过程信息是 LLM 在执行任务过程中的思考过程，包括：

| 类型 | 来源 | 内容 |
|------|------|------|
| 🤔 标签 | LLM 输出 | "用户想要了解...我应该搜索..." |
| 工具调用前思考 | LLM 决策 | "我需要调用 web_search 搜索..." |
| 工具结果分析 | LLM 分析 | "搜索结果显示...我应该总结..." |

#### 4.2 显示方式

**方式一：内联显示（推荐）**

在 AI 回复气泡内，工具卡片上方，显示思考过程：

```
┌─────────────────────────────────────────┐
│ 🦞                                      │
│                                         │
│ 用户想要了解今天的热点新闻...             │  ← 思考过程（浅灰色）
│                                         │
│ > 4 个工具调用 | 6/8 步骤               │
│ 🔧 web_search  完成 0.8s  ▼            │
│ 🔧 read_file   完成 0.1s  ▼            │
│                                         │
│ 根据搜索结果，今天的热点新闻包括...       │  ← 最终回复
│                                         │
└─────────────────────────────────────────┘
```

- 思考过程用浅灰色（#94a3b8），字号 13px
- 与最终回复之间有分隔线
- 默认折叠，点击展开

**方式二：工具卡片内显示**

每个工具卡片内包含该工具调用前的思考：

```
┌─────────────────────────────────────────┐
│ 🔧 web_search  完成 0.8s  ▼            │
├─────────────────────────────────────────┤
│ 参数: {"query": "今日热点新闻"}          │
├─────────────────────────────────────────┤
│ 思考: 用户想要了解...我应该搜索...        │  ← 该工具的思考
├─────────────────────────────────────────┤
│ 结果: 找到 10 条新闻...                  │
└─────────────────────────────────────────┘
```

#### 4.3 🤔 标签处理

- 后端过滤 `<think>...</think>` 标签
- 提取内容作为"思考过程"
- 不直接显示给用户，而是融入思考过程展示

### 五、消息持久化（刷新不丢失）

#### 5.1 后端存储

| 存储位置 | 内容 | 格式 |
|----------|------|------|
| SQLite | 对话元数据（id, title, created_at） | 表 `conversations` |
| SQLite | 消息（id, conversation_id, role, content, tool_calls, thinking） | 表 `messages` |
| 文件 | 完整对话历史（可选） | `memory/sessions/{id}.md` |

#### 5.2 消息表结构

```sql
CREATE TABLE messages (
    id TEXT PRIMARY KEY,
    conversation_id TEXT NOT NULL,
    role TEXT NOT NULL,           -- 'user' / 'assistant' / 'tool'
    content TEXT,                  -- 消息内容
    tool_calls TEXT,               -- JSON: 工具调用列表
    thinking TEXT,                 -- 思考过程
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (conversation_id) REFERENCES conversations(id)
);
```

#### 5.3 加载流程

```
页面加载
    ↓
GET /api/conversations/{id}/messages
    ↓
返回所有消息（含 tool_calls, thinking）
    ↓
前端渲染
    ├── user 消息 → 用户气泡
    ├── assistant 消息 → AI 气泡
    │   ├── thinking → 思考过程（可展开）
    │   ├── tool_calls → 工具卡片（折叠）
    │   └── content → 最终回复
    └── tool 消息 → 工具结果（合并到对应的 assistant 消息中）
```

#### 5.4 刷新后恢复

- 刷新页面后，自动加载最近的对话
- 所有消息（包括工具调用、思考过程）都恢复
- 工具卡片保持折叠状态
- 用户可以展开查看详细信息

### 七、后端处理流程

#### 7.1 消息处理

```python
# 1. 接收 LLM 响应
raw_response = await llm.chat(messages)

# 2. 过滤 🤔 标签
thinking = extract_think_tags(raw_response.content)
clean_content = filter_think_tags(raw_response.content)

# 3. 处理工具调用
tool_calls = []
for tc in raw_response.tool_calls:
    # 执行工具
    result = await tool_registry.execute(tc)

    # 提取有用字段
    useful_result = extract_useful_fields(result)

    tool_calls.append({
        "id": tc.id,
        "name": tc.name,
        "arguments": tc.arguments,
        "result": useful_result,
        "status": "done",
        "elapsed": elapsed_time
    })

# 4. 保存到数据库
save_message(conversation_id, {
    "role": "assistant",
    "content": clean_content,
    "thinking": thinking,
    "tool_calls": tool_calls
})

# 5. 推送给前端
await websocket.send_json({
    "type": "response",
    "conversation_id": conversation_id,
    "message": {
        "role": "assistant",
        "content": clean_content,
        "thinking": thinking,
        "tool_calls": tool_calls
    }
})
```

#### 7.2 提取有用字段

```python
def extract_useful_fields(result: dict) -> dict:
    """去掉 jsonrpc/success/data 外壳，保留有用字段"""
    if isinstance(result, dict):
        # JSON-RPC 格式
        if "result" in result:
            inner = result["result"]
            if isinstance(inner, dict):
                # 去掉 success/data/error 外壳
                if "data" in inner and inner.get("success"):
                    return inner["data"]
                elif "error" in inner:
                    return {"error": inner["error"]}
                return inner
            return inner
        # 普通字典
        return result
    return {"value": str(result)}
```
