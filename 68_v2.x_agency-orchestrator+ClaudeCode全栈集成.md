# Plector + agency-orchestrator + Claude Code 全栈集成（Claude Code 可执行版）

> 目标：
>   用户一句话需求 → Plector 调用 agency-orchestrator 生成工作流 → 调度 186 个专家角色协作 → Claude Code 执行代码开发 → 输出完整代码
>
> 涉及组件：
>   Plector（总控）+ agency-orchestrator（编排）+ agency-agents-zh（角色）+ superpowers-zh（方法论）+ Claude Code（执行）
>
> 共 10 步，按顺序执行。

```yaml
name: "自动开发流水线"
description: "从需求分析到代码实现的全自动开发流程"

agents_dir: "agency-agents-zh"

llm:
  provider: "claude-code"
  model: "claude-sonnet-4-20250514"
  max_tokens: 8192

concurrency: 2

inputs:
  - name: requirement
    description: "开发需求描述"
    required: true
  - name: project_dir
    description: "项目目录路径"
    required: true

steps:
  - id: analyze
    role: "product/product-manager"
    task: |
      分析以下开发需求，输出：
      1. 功能需求清单
      2. 非功能性需求
      3. 验收标准
      4. 优先级排序

      需求：{{requirement}}
    output: requirements

  - id: design
    role: "engineering/engineering-software-architect"
    task: |
      基于以下需求，设计技术方案：
      1. 接口设计（API 端点、请求/响应格式）
      2. 数据库变更（表结构、索引）
      3. 核心逻辑流程
      4. 异常处理策略

      需求：{{requirements}}
    output: tech_design
    depends_on: [analyze]

  - id: security
    role: "engineering/engineering-security-engineer"
    task: |
      审查以下技术方案的安全性：
      1. 权限校验是否完善
      2. 数据泄露风险
      3. 注入攻击防护
      4. 敏感数据处理

      方案：{{tech_design}}
    output: security_review
    depends_on: [analyze]

  - id: implement
    role: "engineering/engineering-senior-developer"
    task: |
      根据以下技术方案和安全审查意见，在项目目录中实现代码：
      1. 严格按照技术方案实现
      2. 遵循安全审查意见
      3. 使用 test-driven-development 技能，先写测试再写实现
      4. 代码风格遵循项目现有规范

      项目目录：{{project_dir}}
      技术方案：{{tech_design}}
      安全意见：{{security_review}}
    output: implementation
    depends_on: [design, security]

  - id: review
    role: "engineering/engineering-code-reviewer"
    task: |
      审查以下实现代码：
      1. 代码质量和可维护性
      2. 是否符合技术方案
      3. 是否遵循安全意见
      4. 测试覆盖是否充分

      实现：{{implementation}}
    output: code_review
    depends_on: [implement]

  - id: summary
    role: "product/product-manager"
    task: |
      汇总开发结果：
      1. 实现了哪些功能
      2. 代码审查意见
      3. 后续改进建议

      需求：{{requirements}}
      实现：{{implementation}}
      审查：{{code_review}}
    depends_on: [review]
```

### 7.3 创建 implementation.py

`skills/auto_developer/implementation.py`：

```python
#!/usr/bin/env python3
"""
自动开发技能 - 从需求到代码的全自动开发流水线

使用 agency-orchestrator 调度专家角色协作，Claude Code 执行代码开发。

Author: Plector
Version: 1.0.0
"""

import subprocess
import logging
from pathlib import Path
from typing import Any

logger = logging.getLogger(__name__)

WORKFLOW_DIR = Path(__file__).parent.parent.parent / "workflows"
AO_OUTPUT_DIR = Path(__file__).parent.parent.parent / "ao-output"


class SkillHandler:
    """自动开发技能处理器"""

    def __init__(self):
        self.name = "auto_developer"

    async def develop(self, requirement: str, project_dir: str = ".") -> dict[str, Any]:
        """
        根据需求自动开发功能

        参数:
            requirement: 开发需求描述
            project_dir: 项目目录路径

        返回:
            {"success": bool, "output": str, "error": str or None}
        """
        workflow_path = WORKFLOW_DIR / "auto_develop.yaml"

        if not workflow_path.exists():
            return {
                "success": False,
                "output": None,
                "error": f"工作流文件不存在: {workflow_path}"
            }

        return await self.run(
            workflow=str(workflow_path),
            inputs={
                "requirement": requirement,
                "project_dir": project_dir
            }
        )

    async def compose(self, description: str) -> dict[str, Any]:
        """
        根据描述自动生成工作流 YAML

        参数:
            description: 工作流描述

        返回:
            {"success": bool, "output": str, "error": str or None}
        """
        cmd = ["npx", "ao", "compose", description]

        try:
            result = subprocess.run(
                cmd,
                capture_output=True,
                text=True,
                timeout=120,
                encoding="utf-8"
            )

            if result.returncode == 0:
                return {"success": True, "output": result.stdout, "error": None}
            else:
                return {"success": False}
        except subprocess.TimeoutExpired:
            logger.error("工作流生成超时")
            return {"success": False, "output": None, "error": "生成超时（2分钟）"}
        except Exception as e:
            logger.exception("工作流生成出错")
            return {"success": False, "output": None, "error": str(e)}

    async def run(self, workflow: str, inputs: dict = None) -> dict[str, Any]:
        """
        运行指定的工作流

        参数:
            workflow: 工作流 YAML 路径
            inputs: 输入变量字典

        返回:
            {"success": bool, "output": str, "error": str or None}
        """
        cmd = ["npx", "ao", "run", workflow, "--quiet"]

        if inputs:
            for key, value in inputs.items():
                cmd.extend(["--input", f"{key}={value}"])

        try:
            result = subprocess.run(
                cmd,
                capture_output=True,
                text=True,
                timeout=600,
                encoding="utf-8"
            )

            if result.returncode == 0:
                # 读取最终输出
                summary = self._read_latest_output()
                return {
                    "success": True,
                    "output": summary or result.stdout,
                    "error": None
                }
            else:
                return {"success": False, "output": result.stdout, "error": result.stderr}
        except subprocess.TimeoutExpired:
            logger.error(f"工作流执行超时: {workflow}")
            return {"success": False, "output": None, "error": "执行超时（10分钟）"}
        except Exception as e:
            logger.exception(f"工作流执行出错: {workflow}")
            return {"success": False, "output": None, "error": str(e)}

    async def plan(self, workflow: str) -> dict[str, Any]:
        """查看工作流执行计划"""
        cmd = ["npx", "ao", "plan", workflow]

        try:
            result = subprocess.run(
                cmd,
                capture_output=True,
                text=True,
                timeout=30,
                encoding="utf-8"
            )

            return {
                "success": result.returncode == 0,
                "output": result.stdout,
                "error": result.stderr if result.returncode != 0 else None
            }
        except Exception as e:
            logger.exception(f"工作流计划查看出错: {workflow}")
            return {"success": False, "output": None, "error": str(e)}

    async def list_roles(self) -> dict[str, Any]:
        """列出所有可用的 AI 角色"""
        cmd = ["npx", "ao", "roles"]

        try:
            result = subprocess.run(
                cmd,
                capture_output=True,
                text=True,
                timeout=30,
                encoding="utf-8"
            )

            return {
                "success": result.returncode == 0,
                "output": result.stdout,
                "error": result.stderr if result.returncode != 0 else None
            }
        except Exception as e:
, "output": result.stdout, "error": result.stderr            logger.exception("角色列表获取出错")
            return {"success": False, "output": None, "error": str(e)}

    async def list_workflows(self) -> dict[str, Any]:
        """列出所有可用的工作流模板"""
        cmd = ["npx", "ao", "list_workflows"]

        try:
            result = subprocess.run(
                cmd,
                capture_output=True,
                text=True,
                timeout=30,
                encoding="utf-8"
            )

            return {
                "success": result.returncode == 0,
                "output": result.stdout,
                "error": result.stderr if result.returncode != 0 else None
            }
        except Exception as e:
            logger.exception("工作流列表获取出错")
            return {"success": False, "output": None, "error": str(e)}

    def _read_latest_output(self) -> str | None:
        """读取最新的 ao-output 输出"""
        if not AO_OUTPUT_DIR.exists():
            return None

        dirs = sorted(
            [d for d in AO_OUTPUT_DIR.iterdir() if d.is_dir()],
            key=lambda d: d.stat().st_mtime,
            reverse=True
        )

        if not dirs:
            return None

        summary_file = dirs[0] / "summary.md"
        if summary_file.exists():
            return summary_file.read_text(encoding="utf-8")

        return None
```

# auto_developer - 自动开发流水线

## 功能

从需求到代码的全自动开发流水线。使用 agency-orchestrator 调度 186 个专家角色协作，Claude Code 执行代码开发。

## 工具

### develop

根据需求自动开发功能。

参数：
- requirement: 开发需求描述（必需）
- project_dir: 项目目录路径（可选，默认当前目录）

示例：
```
自动开发：给用户模块增加批量导出 CSV 功能，要求分页、权限校验、异步处理
```

### compose

根据描述自动生成工作流 YAML。

参数：
- description: 工作流描述（必需）

### run

运行指定的工作流。

参数：
- workflow: 工作流 YAML 路径（必需）
- inputs: 输入变量字典（可选）

### plan

查看工作流执行计划（DAG）。

### list_roles

列出所有可用的 AI 角色（186 个）。

### list_workflows

列出所有可用的工作流模板（30 个）。

## 工作原理

```
用户需求 → compose 生成 YAML → run 执行工作流 → 输出代码
```

工作流内部：
1. 产品经理分析需求
2. 架构师设计技术方案（并行）+ 安全工程师审查（并行）
3. 高级开发者实现代码
4. 代码审查员审查
5. 产品经理汇总

## 配置要求

- Node.js 18+
- agency-orchestrator（npm install -g agency-orchestrator）
- Claude Code CLI（或 DeepSeek/OpenAI API Key）
```

## 组件职责

| 组件 | 职责 | 说明 |
|------|------|------|
| Plector | 总控 | 接收需求、调用 MCP、管理上下文和记忆 |
| agency-orchestrator | 编排 | 生成工作流 YAML、调度角色、DAG 并行执行 |
| agency-agents-zh | 角色 | 186 个专家角色的 system prompt |
| superpowers-zh | 方法论 | TDD、代码审查、调试等编程方法论 |
| Claude Code | 执行 | 底层代码生成和文件操作 |

## 调用链路

```
用户: "给用户模块加批量导出功能"
  ↓
Plector: 调用 auto_developer.develop()
  ↓
agency-orchestrator: compose 生成 YAML
  ↓
agency-orchestrator: run 执行工作流
  ├── 产品经理: 分析需求 → 输出规格
  ├── 架构师: 设计接口 → 输出方案 (并行)
  ├── 安全工程师: 审查权限 → 输出意见 (并行)
  ├── 高级开发者: 实现代码 → 写入文件
  ├── 代码审查员: 审查代码 → 输出意见
  └── 产品经理: 汇总结果 → 输出 summary
  ↓
Plector: 读取 ao-output/summary.md
  ↓
用户: 收到完整结果
```
